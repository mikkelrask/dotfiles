Custom Neovim installer

Since the version of neovim that are bundled in most official linux repos are of a certain date, and we always want to make sure to have the latest version, I've made this install script that simply fetches the latest build from neovim's repos on Github.

If previous installs or leftovers from previous installs are detected, these will be purged, unless you are using the interactive mode, where you will be prompted on what to do.

Latest vs Nightly‚Äã

The installer can get either the package tagged with latest stable or the nightly development builds. Unless otherwise specified (through flags) the installer will default to the stable version.

Flags‚Äã

When making these type of utils, I often think about those philosofies that unix/linux in general are build up around. Less is more, but with modularity in mind. Always rather make a utility that does one thing, and one thing good, than a huge confusing suite of functions, built into one file.

While it's not always easy, I try to make the utils I create hands free in it's basic form, so to speak. This is the modularity part of the philophy that wins, so I can use one util to call another, wihtout having to gather user input for example But that also means you have to make decissions on behalf of any potential users who will end up using it. And I hate that part, and love freedom and customization options!
So while the utils are made to be automatic, the user can instead make their own by passing the correct flags, for their custom neovim install, for example.

Interactive mode‚Äã

Interactive install
./install_nvim.sh --interactive
# or it's shorthand
./install_nvim.sh -i

This will prompt you for all the choices the installer offers. If leftovers from previous installs are found, you will be prompted if you wish to purge these, where the installer otherwise is made to be "hands free", and assume some sane defaults. (if stuff is found purge it, install the new stable stuff and the dots)

Get the Nightly build‚Äã

Install the development branch version
./install_nvim.sh --nightly
# or it's shorthand
./install_nvim.sh -n

This is the only thing the flag does - the rest will still be assumed!

Get the latest stable build‚Äã

Install the build tagged with Latest
./install_nvim.sh --latest
# or it's shorthand
./install_nvim.sh -l

This is the only thing the flag does - the rest will still be assumed!

Install path?‚Äã

Where to install nvim
./install_nvim.sh --dir /YOUR/DESIRED/PATH/
# or it's shorthand
./install_nvim.sh -d /YOUR/DESIRED/PATH/

Only the nvim file itself will be placed in the selected path, where the shared and state directories are still going to be located in $HOME/.local and the dotfiles located in according to your $DOTFILES_HOMEDIR, if set. This is assumed as $HOME/dotfiles otherwise.

Cleanup‚Äã

Purge neovim and related folders
./install_nvim.sh --purge
# or it's shorthand
./install_nvim.sh -p

If you for whatever reason wan't to delete neovim in its entirety, and what has been installed either by the installer itself or by you, adding the -p-flag will do this for you. Combined with the -i flag, you will be prompted to confirm each directory/file that is set to be deleted.

But what is exactly getting purged? ü§î‚Äã

Glad you asked!

$HOME/
    - .cache/nvim
    - .local/bin/nvim
    - .local/state/nvim/
    - .local/share/nvim/
    - .config/nvim/

Just as it will unistall neovim, if its found to be installed by your systems package manager/found in /bin/, /usr/bin/ or /usr/local/bin/. It will use the default package manager for this action!

Keep‚Äã

Keep downloaded files
./install_nvim.sh --keep
# or it's shorthand
./install_nvim.sh -k
See help message
./install_nvim.sh --help
# or it's shorthand
./install_nvim.sh -h

And of course all flags can be used individually or combined, though most of the combinations would be somewhat contradicting, but will work if you insist.

Combining passed flags
# Install the Nightly development build and keep the downloaded files
./install_nvim.sh -nk 
# "Stacked" arguments will be executed one at a time
./install_nvim.sh -p -n -l -d /bin/

The last example would result in -p purging everything that currently exists, and then the -n gets you the latest nightly build as usual, and then the -l will fetch the latest stable nvim and place that in the directory that where specified with the -d option.
Basically the function that is exectued by any flag (i.e purge) will before executing look at argument next in line, and if that flag can't be "combined" with itself, the current flag will just execute, and move along to the next flag, that will do the same check, until we are at the end.

Code‚Äã

Install the latest Neovim
#/bin/bash
cd /tmp
URL=''
ANS=''
VERSION=''
err () {
  echo "Ôäç Error!" 
  echo "Something went wrong: $1"
  echo "Press any key to exit installer"
  read -n 1 BYE
  exit 1
}
prompt_purge () {
  if [ -f "$HOME/.local/bin/nvim" ] || [ -d "$HOME/.local/share/nvim" ] || [ -d "$HOME/.local/state/nvim" ] || [ -d "$HOME/.local/lib/nvim" ] || [ -d "$HOME/.cache/nvim" ]; then
    echo "Do you wish to purge files from previous neovim installs? (y/n) "
    read ANS
    ANS=${ANS:-'n'}
    if [ "$ANS" = 'n' ] || [ "$ANS" = 'N' ]; then
      echo "ÔÅ° Skipping purge"
    else
     purge 
    fi
  fi
}
purge () {
  rm -rf "$HOME/.local/bin/nvim" "$HOME/.local/share/nvim" "$HOME/.local/lib/nvim" "$HOME/.local/state/nvim" "$HOME/.cache/nvim" || err "Ôäç Could not purge leftovers"
  echo "Ôá∏ Leftovers are purged"

}

prompt_for_version () {
  echo "Neovim Downloader by mikkelrask"
  echo " - Pick your version below (enter number)"
  echo " - 1. Latest (stable)"
  echo " - 2. Nightly (development / prerelease)"
  echo ""
  echo "Enter number: "
  read ANS

  if [ "$ANS" -eq 1 ]; then
    URL='https://github.com/neovim/neovim/releases/latest/download/nvim-lDoneinux64.tar.gz'
    VERSION="latest stable"
  fi
  if [ "$ANS" -eq '2' ]; then
    URL='https://github.com/neovim/neovim/releases/download/nightly/nvim-linux64.tar.gz'
    VERSION="nightly"
  fi
  echo "ÔÄô downloading $VERSION build ..."
}
unpack_tar () {
  # check if tar exists
  if ! [ -x "$(command -v tar)" ]; then
    err "Ôäç tar is not installed"
  fi
  tar -xf nvim.tar.gz || err "ÔÅ¶ unpacking tar archive failed"
  echo "ÔÅ¶ unpacked tar archive" 
}
copy_nvim () {
  cd nvim-linux64
  cp -r bin share lib $HOME/.local/  || err "ÔÉÖ copying files failed"
  echo "ÔÄå nvim installed in .local/bin"
}
get_nvim () {
  wget -O nvim.tar.gz $URL -q --show-progress || err "Ôäç could not download Neovim $VERSION build."
  echo "ÔÄå nvim-linux64.tar.gz downloaded"
  unpack_tar 
  prompt_purge
  copy_nvim
  if [ "$KEEP" ]; then
    echo "ÔÅ° Keeping downloaded files"
  else
    prompt_clean
  fi
  echo "ÔÄå Neovim installed successfully!"
}
clean_up () {
  rm -rf /tmp/nvim-linux64 /tmp/nvim.tar.gz
  echo "ÔÄå Cleaned up temporary files"
}
prompt_clean () {
  echo "Do you wish to clean up downloaded files? (y/n) "
  read ANS
  ANS=${ANS:-'y'}
  if [ "$ANS" = 'n' ] || [ "$ANS" = 'N' ]; then
    echo "ÔÅ° Skipping cleanup"
  else
    clean_up
  fi
}
if [ -z "$1" ]; then
  NOARGS=1
else
  if [ "$1" = "-k" ] || [ "$1" = "--keep" ]; then
    KEEP=1
  elif [ "$1" = "-p" ] || [ "$1" = "--purge" ]; then
    echo "Ôá∏ Purging previous neovim installs and leftovers"
    echo "This action can not be undone. Continue? (y/n) "
    read ANS
    ANS=${ANS:-'n'}
    if [ "$ANS" = 'n' ] || [ "$ANS" = 'N' ]; then
      echo "ÔÅ° Skipping purge"
      exit 0
    fi
    purge
    exit 0
  else
    echo "Usage: install-nvim.sh [OPTION]"
    echo "This script will download and install neovim in .local/bin"
    echo "-k or --keep will keep the downloaded files"
    echo "-p or --purge will remove any previous neovim installs"
    exit 0
  fi
fi

prompt_for_version || err "something went wrong..."
get_nvim 
prompt_clean
